Nominal Parameters – Descriptions and Questions

 

 

General information

Each parameter can be associated with a set of existential assertions, each of the type “in the language L, there is an observable grammatical object (construction/morpheme/feature/etc.) a with property x”. Each assertion is questioned as to its truth value: “Is the assertion true?”. The existence of the grammatical object a (i.e., the answer YES) can be assumed on the basis of positive evidence only. In turn, only an answer YES can set a parameter’s value to +.

In principle, just one answer YES for each parameter is enough to set the value of that parameter. Thus, if no question receives an answer that unambiguously determines the value of a certain parameter in a language, the supposed default value of that parameter is chosen. Since + is always considered to be determined on the basis of positive evidence, it follows that in parameters with a default value, that value can only be –.

Some statements also include a set of conditions that must be met for the relevant question to apply meaningfully (preconditions), e.g. other constructions/morphemes/features/etc.

 

Instructions – PLEASE, read carefully before answering the questions

 

1. The present file contains, for each parameter, the following information:

a. Description: a short description of the parameter

b. Implication(s): a description of its implications

c. Questions: a list of questions which set the parameter value. Each question has a label, that contains the number of the relevant parameter (P1_, P2_, P3_ etc.) and that of each single question (P1_Qa, P1_Qb, etc.): these labels are identical to those found in the Excel file (see point 2, below).

d. Example: for every question, (at least) one example for each of the answers YES

e. Manifestation clustering: a list of propositional formulas expressing the connection among the various manifestations of the same parameter.

 

2. The excel file (labeled ANSWERS_X, where X is the name of a language) contains a list of the 91 parameters and their questions. In particular:

a. the green-shaded lines correspond to the 91 parameters (and their numbers, implications and labels).

b. the non-shaded lines correspond to the question.

c. 1st column: progressive number of the questions.

d. 2nd column: progressive numbers of the parameters (P1, P2, P3, …) and labels for each question (P1_Qa, P1_Qb etc.). The latter correspond to the label assigned to each question in the present file.

e. 3rd, 4th and 5th column: name, implications and label of each parameter, respectively.

f. 6th column: short name of the question

g. 7th column: value of the parameter (+/-/0), in the green-shaded cell; answer to the question (YES/NO) in the non-shaded cells.

h. 8th, 9th, 10th column: example(s), glosses, translation

i. 11th column: comments and other relevant information

 

3. How to proceed. For each question:

a. answer it (YES or NO). If the question cannot be answered because the language does not meet the precondition(s) for that question, please, answer NO* and add a comment in the relevant column (the question does not apply because…).

b. provide an example for each YES value and, if the question has a precondition, add a further example that shows that the precondition is met.

c. you can also provide examples of ungrammatical sequences/structures (if your language does not answer YES to a given question).

d. for each example, add a gloss and the translation(s).

e. add (in the comment column) any relevant information that could help understand the empirical manifestations of the relevant question in your language.
